## 한줄 요약
Style Reference와 Image Prompt의 차이를 이해하는 것이 AI 이미지 생성 품질의 핵심이며, Replicate API로 후처리 파이프라인을 자동화하면 프로덕션 수준의 에셋 생산이 가능하다.

## 핵심 포인트
- Midjourney: 레퍼런스 이미지 활용 시 성능 대폭 향상, 디자이너 스타일 레퍼런스와 조합으로 일관된 아웃풋 가능
- Style Reference vs Image Prompt 핵심 차이: Style은 색감/분위기 우선, Image Prompt는 구도/구성 우선 — 동시 사용 시 의도치 않은 결과 빈발
- AI 기반 텍스트 삽입은 신뢰도 여전히 낮음 — 실무에서 주의 필요
- 실무 워크플로우: 4개 옵션 생성 → 베이스 선택 → 후처리 3단계
- Figma + Gemini 이미지 에디팅 연동: 선택 영역만 편집, 객체 제거/수정을 Figma 내에서 직접 처리
- AI Isolator: 단일 이미지에서 개별 레이어 자동 분리 → 스크롤 패럴랙스·애니메이션에 활용
- 어두운 배경 제거 팁: 검정 배경 제거 어려움 → 빨간 배경 사용 권장
- Replicate API 파이프라인: 해상도 향상(Upscaling) → 프레임 분리 → 배경 제거 → 전체 자동화
- 최종 아웃풋은 WebM 포맷 변환 필수
- SVG 디자인/애니메이션은 오픈소스 코드베이스 활용 권장

## 전문 요약
여준호(스트로크) 발표. AI 이미지 생성 도구 및 프로덕션 워크플로우 공유.

**Midjourney 변화**: 레퍼런스 이미지 활용 시 실무 수준의 아웃풋 가능. 디자이너 스타일 레퍼런스와 조합하면 일관성 확보. 3D 에셋 생성도 가능하나 프롬프트 재현이 어렵고 수동 튜닝 여전히 필요.

**Style Reference vs Image Prompt**: 가장 중요한 구분. Style Reference는 색감/분위기를 우선시해 원본 이미지와 충돌 가능. Image Prompt는 구도/구성 우선. 두 개를 동시에 쓰면 의도치 않은 결과가 빈번하게 발생. 실무 워크플로우는 4개 생성 → 베이스 선택 → 후처리.

**Figma + AI 연동**: Gemini 이미지 에디팅으로 Figma 내에서 선택 영역만 편집 가능. 객체 제거/수정이 별도 툴 없이 처리됨. AI Isolator로 단일 이미지에서 레이어 자동 분리 → 각 요소별 독립 모션 컨트롤 가능. 스크롤 패럴랙스·애니메이션 구현에 직접 활용.

**후처리 자동화**: Replicate API로 Upscaling → 프레임 분리 → 배경 제거 파이프라인 자동화. 배경 제거 시 검정 대신 빨간 배경 권장. 최종 출력은 WebM 변환 필수. SVG는 오픈소스 코드베이스 활용.
