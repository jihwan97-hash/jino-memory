## 한줄 요약
네이티브 툴 콜링을 우회하고 텍스트 기반 미들웨어로 모델별 최적화를 하면 Berkeley Function Calling 벤치마크에서 70점→95점(35% 향상)이 가능하다.

## 핵심 포인트
- Hashline 코드 에디팅: 라인별 해시 생성 후 라인넘버-해시 페어를 앵커로 활용 → DB의 Compare-and-Swap(CAS) 개념 적용, 페일률 획기적 감소
- 오픈소스 자연 유입 46개 스타 + Vercel AI SDK 공식 문서 커뮤니티 미들웨어 섹션 등재
- 모델별 채팅 템플릿(Jinja)이 다르며 대부분 비공개 — 네이티브 툴 콜링은 모델별 특화 학습 결과
- 미들웨어 구현: 툴 스키마를 시스템 프롬프트에 직접 렌더링, 모델에게 순수 텍스트만 전송
- 모델별 포맷 최적화: Claude는 XML 선호, GPT는 JSON 선호 — 이 차이를 미들웨어가 반영
- Berkeley Function Calling 벤치마크 70점 → 95점 달성 (35% 향상)
- 툴 콜링 미지원 모델(Gemma3 등)에서도 동작 가능 — 지원 모델들도 추가 향상
- Guided Generation(Structured Output): 토큰 생성 시 로짓 필터링으로 스키마 준수 강제
- 리즈닝 모델 등장으로 Structured Output 성능 저하 문제가 완화되는 추세
- 터미널 벤치 테스트용 에이전트 하네스 + 한국어 자동 번역 기능 포함 개발

## 전문 요약
민웅기(FriendliAI) 발표. AI 툴 콜링 미들웨어 프로젝트 소개.

**Hashline 에디팅**: 기존 placeholder 방식을 개선한 해시 기반 앵커 시스템. 각 라인의 두 자릿수 해시를 생성 후 라인넘버-해시 페어로 코드 에디팅의 신뢰성을 높임. DB의 Compare-and-Swap과 유사한 개념으로 컨텐츠 일치 시에만 편집 유효. 오픈소스로 공개해 자연 유입 46개 스타를 얻었고 Vercel AI SDK 공식 문서 커뮤니티 섹션에 등재됨.

**미들웨어 구조**: 각 프로바이더의 채팅 템플릿 렌더링 내부 구조(Jinja 기반)는 대부분 블랙박스. 이를 우회하여 툴 스키마를 시스템 프롬프트에 직접 렌더링 → 모델에게 순수 텍스트만 전송. Claude는 XML, GPT는 JSON 선호도를 반영한 커스터마이징 가능.

**성능**: Berkeley Function Calling 벤치마크 70→95점(35% 향상). 툴 콜링 미지원 모델에서도 동작. Guided Generation(로짓 필터링)으로 100% 포맷 준수 강제 가능.

**실제 적용**: 터미널 벤치 테스트 에이전트 하네스 개발, 한국어 자동 번역 포함. 네트워크 캡처 기반 API 호출 분석 도구 활용. Claude Code 동시성 기반 차단 우회 방법도 논의됨.
