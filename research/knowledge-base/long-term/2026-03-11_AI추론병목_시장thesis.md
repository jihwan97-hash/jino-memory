## 1장. 병목의 물리학: 왜 메모리가 한계인가

### 훈련 vs 추론 구조 차이
- **훈련**: 동일 가중치에 수천 개 배치 병렬 처리 → GPU 대규모 병렬 코어 최적
- **추론**: 토큰 순차 생성. 매 토큰마다 전체 가중치를 메모리에서 로드

### 메모리 규모 문제
- GPT-4급(~1.8조 파라미터) BF16 → **약 3.6TB 메모리 필요**
- H200 HBM3E 용량: **141GB** → 수십 개 GPU NVLink 분산 추론 필수

### 핵심 지표: 산술 강도 (Arithmetic Intensity)
- LLM 추론 = **Memory-Bound** 워크로드 (FLOP per byte 낮음)
- FLOPS가 아니라 **메모리 대역폭(GB/s)** 이 성능 결정

| 칩 | HBM | 용량 | 대역폭 |
|---|---|---|---|
| H100 | HBM3 | 80GB | 3.35 TB/s |
| H200 | HBM3E | 141GB | 4.8 TB/s |
| Rubin R100 | **HBM4** | **288GB** | **22 TB/s** |

> Rubin = Memory Wall 자체를 허무는 시도. Blackwell 대비 대역폭 2.8배↑

**역설**: HBM 용량/대역폭 증가 → 모델도 더 커짐 (파라미터↑, 컨텍스트↑, KV 캐시↑) → 메모리는 항상 다음 세대의 병목

## 2장. KV 캐시 — 추론을 지배하는 숨겨진 메모리 괴물

- Transformer Attention = 이전 모든 토큰 "기억" → 그 실체가 **KV 캐시**
- 컨텍스트 길이↑ → KV 캐시 폭발적 증가 (100만 토큰 = 수십~수백 GB, 모델 가중치보다 커질 수 있음)
- **KV 캐시 병목 = 서비스 수익화의 직접적 상한선** (동시 처리 요청 수 제한)

### NVIDIA ICMS (Inference Context Memory Storage Platform)
- CES 2026 공개, BlueField-4 DPU 활용
- 기존 대비: 토큰 처리량 5배, TCO 대비 성능 5배, 전력효율 5배, TTFT **20배** 개선
- Vera Rubin SuperPod: GPU당 최대 16TB 컨텍스트 메모리, 단일 SuperPod 최대 **18,432TB**

## 3장. HBM 공급망 — 병목의 물리적 실체

### 시장 규모
- 2025년 $38B → 2026년 **$58B** 성장 예상
- 점유율: SK Hynix 62% / Micron 21% / Samsung 17%

### 수요
- 2025년 HBM 수요 YoY **+130%**, 2026년도 **+70%** 예상
- NVIDIA Blackwell(B300, GB300), Google TPU, AWS Trainium의 HBM3e 전환이 동력

### 공급 제약 (완판)
- **SK Hynix CFO**: "2026년 전체 HBM 물량 이미 완판"
- **Micron CEO**: "2025~2026년 HBM 캐파 완전 예약"
- **TSMC CoWoS**: 2026년 중반까지 완판

### HBM 제조의 구조적 어려움
- TSV(Through-Silicon Via) 수직 적층 공정 = 단기 증설 불가
- 밀도 패널티: HBM3 = 0.16 Gb/mm² vs DDR4 = 0.296 Gb/mm² → 동일 면적에서 훨씬 적은 비트

### 가격
- 2025 Q4: 서버 DDR5 대비 HBM3e 프리미엄 **4~5배** → 2026 말 **1~2배** 축소 전망
- (이유: HBM 저렴해지는 게 아니라 DDR5 가격이 동반 상승하기 때문)

## 4장. SRAM vs HBM — 아키텍처 전쟁 (Groq 인수 맥락)

| 항목 | Groq LPU (SRAM) | NVIDIA GPU (HBM) |
|---|---|---|
| 대역폭 | **80 TB/s+** | ~8 TB/s |
| 지연 | 극저 (정적 스케줄링) | 동적 스케줄링 비결정적 지연 |
| 용량 | **230MB/칩** | H200 141GB, Rubin 288GB |
| Rubin 대비 | Rubin 1개 = LPU **1,280배** 용량 | — |

- **NVIDIA의 Groq $20B 인수 (2025.12)**: GPU 원시 처리능력만으로 부족 인정. non-exclusive 기술 라이선싱 구조
- 2026년 Vera Rubin = GPU + LPU 하이브리드 설계 예상

## 5장. 차세대 메모리 계층 지도

| 계층 | 기술 | 용량 | 대역폭 | 용도 |
|---|---|---|---|---|
| 1계층 | SRAM (LPU) | 수백 MB | 80 TB/s | 극저지연 |
| 2계층 | HBM4/4E | 288GB~ | 22 TB/s | GPU 온보드 |
| 3계층 | HBF/DRAM | 수 TB | ~1.6 TB/s | KV 캐시 확장 |
| 4계층 | NVMe SSD (ICMS) | 수십 PB | — | KV 캐시 오프로드 |

### SK Hynix HBF (High-Bandwidth Flash)
- 2026년 공개 예정
- 용량 **512GB** (HBM4 64GB의 8배), 대역폭 **1,638 GB/s+**
- 읽기 전용 모델 가중치 → Flash 오프로드, HBM은 KV 캐시에 집중

> David Patterson (Turing Award) + Google 공동 논문: "LLM 성능의 진짜 과제는 FLOPS가 아니라 메모리와 인터커넥트"
