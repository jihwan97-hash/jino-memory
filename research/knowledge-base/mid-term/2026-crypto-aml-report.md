## 1. 개요 — AML 역사와 가상자산

- 자금세탁 어원: 알 카포네의 세탁소·카지노 위장사업 (금주령 시대 1920~1933)
- 당시 자금세탁 자체는 범죄 아님 → 알 카포네는 탈세로 기소
- 국제 범죄화: 1988년 UN 비엔나 협약 → FATF 설립(1989) → 9.11 이후 테러자금조달금지법 추가
- 가상자산 = 현대판 알코올: 중국이 2018년 전면 금지했지만 홍콩 등 역외로 우회만 됨
- 고위험 국가: UAE, 튀르키예, 북키프로스, 에스토니아, 캄보디아
- 한국: FATF "정규 후속점검" 최고 등급 — 하지만 리스크 0이 아님, 지속적 대응 필요

## 2. 예측 시장 (Prediction Markets) — AML 리스크

### 핵심 명제
**"블록체인이 투명해도 AML과 다른 문제"**
- 온체인은 거래의 존재와 정산 결과의 무결성만 증명
- 어떤 정보 환경, 이해관계에서 거래됐는지는 증명 불가
- 정산 완료 후 모든 수익 = "합법적 예측 수익"으로 설명 가능 → 검증 요구 없는 구조

### 시장 규모
- 현재 약 $10B 거래량, 지속 성장 중 (Polymarket 등)

### 자금세탁 3대 시나리오
1. **스머핑(Smurfing)**: 불법 자금을 다수 지갑으로 쪼개 각각 다른 베팅 → 새 지갑 생성 비용 0에 수렴
2. **워시 트레이딩**: 불법 자금 A 포지션 + 깨끗한 돈 반대 포지션 → 합법적 트레이딩 수익 위장. 예측시장 특성상 FDS 탐지 기준 적용 어려움
3. **내부자 거래/뇌물**: 정보 비대칭성 이용 → 정교한 세팅 없이도 가능 = 가장 큰 취약점

### 구조적 취약점
- 거래 대상이 자산이 아닌 **사건** → 가격 급변이 "자연스러운 현상"으로 해석 가능
- 손익 구조 명확(0 또는 1) → 손실/수익 모두 사건 결과로 일관되게 설명 가능
- KYC 없는 비수탁 지갑 연결 → 트래블 룰 적용 불가

### 전통 AML이 예측시장에서 실패하는 이유
1. **기준선 설정 불가**: 가격 변동성이 본질적으로 크기 때문에 "정상 범위 → 이탈 탐지" 논리 붕괴
2. **공정성/조작 리스크**: 시장 생성 단계(사건 선정·정의)에서 조작 가능, 온체인 기록 없음
3. **CDD 주체 공백**: 비수탁 지갑 → VASP 식별 기술적 불가 → 의무 수행 중재자 없음

### 플랫폼별 규제 준수 현황
| 플랫폼 | 특징 | 규제 수준 |
|--------|------|-----------|
| **Kalshi** | CFTC DCM 라이선스, Robinhood·Coinbase 제휴, 중앙화 오더북 | 높음 |
| **Polymarket** | Polygon 기반 스마트컨트랙트, KYC 없음, 글로벌/US 분리 운영 | 부분적 (미국만) |
| **Opinion** | BNB체인, KYC 없음, Pre-TGE 이벤트 베팅 특화 | 낮음 |

- 폴리마켓 2026.02.03 네바다주 임시 제한 명령
- 한국: 직접 규제 없으나 법적으로 불법 도박 판단

### AML 재설계 방향
1. **시장 생성 단계** 위험 통제 (FATF 권고 15: 새로운 기법 사전 평가)
2. **오라클/판정 구조** 독립적 리스크 평가
3. **외부 정보 환경** (언론·SNS·여론) 구조적 위험 요인으로 평가
4. **온체인 거래 증빙 인프라** 구축 (개인 투자자 소명 부담 완화)
5. 예측시장 사업자를 **DNFBP(특정 비금융 사업자)** 로 규정 검토

## 3. 크립토 카드 페이먼트 — AML 리스크

### 시장 현황
- 2025년 스테이블코인 온체인 전송 규모 $30~40조 → Visa/Mastercard 수준 도달
- Visa와 Mastercard 2025년 4월 스테이블코인을 차세대 결제 핵심으로 선정

### 크립토 결제 3유형 (규제 접점 기준)
| 유형 | 예시 | 전통 금융 접점 |
|------|------|--------------|
| 온체인 네이티브 | 스테이블코인 P2P | 없음 |
| 크립토-크립토 브릿지 | XRP, Stellar, Solana Pay | 양 끝단만 |
| **크립토-법정화폐 브릿지** | 크립토 카드, On/Off-Ramp | 전면 적용 |

### 크립토 카드 작동 구조 (Kast 사례)
1. KYC 후 외부 지갑 → Kast 계정으로 USDC/USDT 입금
2. Elliptic이 실시간 자금 출처 분석 (오염 주소 차단)
3. Sumsub이 신원 확인
4. 결제 시 Just-in-Time 방식으로 스테이블코인 → 법정화폐 변환
5. Visa 레일 통해 가맹점 정산

### 핵심 AML 이슈
- **"진입점" 원칙**: FATF 2015년 가이드라인 — 전통 금융 진입점에 AML 집중
- Off-Ramp 순간 = 전통 금융 진입 → 자금 출처 검증 필수
- **소명 구조적 한계**: 크립토 트레이딩 수익의 경우 은행 명세서·급여 명세서로 소명 불가
  - Kast 동결 사례: perp-dex 수익 → 외부 출금 시도 → 동결 → 은행 명세 요구 → 소명 수단 없음

### 이해관계자 책임 분배
- 카드 발급 플랫폼 = 모든 규제 책임 집중 (가맹점은 크립토 관여 여부 불인지)
- 블록체인 분석 파트너 (Elliptic 등) 실시간 모니터링

## 4. 프라이버시 코인 & 툴 (Privacy Coins and Tools)

### 시장 현황
- 2025년 하반기 Zcash(ZEC) 급등, Shielded Pool 볼륨 대폭 증가
- 비탈릭 부테린이 이더리움 프라이버시 프로토콜 **Railgun** 공개 지지 + $260만 투자
- 가장 널리 쓰인 믹서: **토네이도 캐시** (zk-SNARKs 기반)

### 토네이도 캐시 법적 쟁점
- 2022년 OFAC 제재 리스트 추가 → 2024년 미국 법원 "국제비상경제권한법 위반" 판결
- 법원 핵심 논리: 스마트 컨트랙트 코드 = property 아님 (불변성, 귀속성 없음)
- non-custodial: 자산에 대한 통제권이 사용자에게만 있음 → money transmitter 해당 안 됨
- 창립자 로만 스톰: 자금세탁 공모죄 기소 (고의성 쟁점 — 의견 엇갈림)

### 대표 AEC(익명화 강화 코인) 비교

| | Zcash (ZEC) | Monero (XMR) |
|---|---|---|
| 기술 | zk-SNARKs (대수 기하학) | Ring Signature (선형 대수) |
| 익명화 강도 | 기술적으로 더 강력 | Ring 축소 시 탈 익명화 가능 |
| 선택적 공개 | Viewing Key 제공 가능 | 불가 |
| 거래소 상장 | 투명 주소만 취급 → 실질 프라이버시 기능 제한 | 대부분 상장폐지 |
| 규제 친화성 | 상대적으로 높음 | 낮음 |

- IRS: 모네로+라이트닝 네트워크 추적 성공 툴 개발자에게 $625,000 포상금 현상
- 모네로 커뮤니티: FCMP++ 연구 중 (Ring 크기 수백만 개로 확대해 탈 익명화 방지)

### 규제 준수 가능한 프라이버시: Zcash 모델
- 거래소: 투명 주소(t-addr)만 취급
- 개인이 은닉 주소(z-addr)로 내부 세탁 후 투명 주소로 전환 시 → Viewing Key 제출 요구 가능
- 한계: Shielded Pool 사용 자금 → 바이낸스 등이 고위험군 지정

### 핵심 AML 딜레마
- VASP의 프라이버시 코인 상장폐지 → 규제 시야 밖으로 밀어내는 역설
- 금지 ≠ 자금세탁 차단, 오히려 지하 경제로 이동
- 진짜 기준: "사후적으로라도 거래 정당성 검증 가능한 구조인가?" + "책임 귀속 가능한 주체가 있는가?"

### 규제 방향 제언
1. **VASP**: on/off ramp 구간 거래 모니터링 강화, 프라이버시 코인 통일 기준 마련
2. **CTR 적용**: 현재 거래량 수준에선 전면 자동보고 비효율 → RBA + 트래블룰로 자동 기록·보관이 현실적 대안
3. **조건부 허용**: 선택적 정보 공개, 악용 자금 분리, 규제 대응 인터페이스 갖춘 프라이버시 프로토콜은 제도권 내 관리 가능
4. **'스마트 규제'**: 혁신 억압 아닌 악용만 정밀 차단

## 5. 결론

- AML의 핵심 = **협력**: 국가 간 + VASP ↔ FIU 소통
- 예측시장, 크립토 페이먼트, 프라이버시 코인 모두 "중개자 중심 통제 구조" 약화 방향으로 진화
- 단순 확장이 아닌 **구조적 재설계** 필요
- AML = 비용이 아닌 **시장 신뢰 확보를 위한 전략적 투자**

## 투자/컴플라이언스 시사점

- **Polymarket** 등 예측시장 연동 지갑: 한국에서 리스크 평가 기준 필요
- **크립토 카드** 소명 인프라: 온체인 거래 내역을 전통 금융 서류 대체 가능한 형태로 구조화 수요
- **DNFBP 규제 확장**: 예측시장 사업자·분석 기업·컴플라이언스 솔루션 업체가 AML 인프라 주체로 부상
- **Kalshi 모델**: 규제 친화적 예측시장이 기관 수요 흡수 가능 — 한국 유사 서비스 인허가 논의 시 참고
