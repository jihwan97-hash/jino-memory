# Memory Retrieval Protocol v1
> Inspired by @NathanYJLee (Factomind) — 2026-03-04

## 핵심 원칙
기억 검색은 비용이 있다. 필요할 때만, 최소한으로.

---

## Fast-path (기본 — retriever 호출 안 함)
아래 해당 없으면 그냥 답변. memory_search 스킵.
- 명확한 일반 질문 (날씨, 정의, 계산 등)
- 방금 나온 컨텍스트 안에 답이 있는 경우
- 새 작업 지시 (파일 저장, 일정 생성 등)

---

## Slow-path 트리거 (아래 중 하나라도 해당 시 memory_search 실행)

### 유저 전용 시그널 (키워드)
- "우리", "예전에", "저번에", "설정", "정책"
- "기억해?", "리마인더", "크론", "맞지?", "그거"
- "형 선호", "내 스타일", "우리가 했던"

### 불확실성 시그널
- 내 답변 초안에 "아마", "가능성", "~일 수도" 등장
- 사실 확인이 중요한 컨텍스트 (날짜, 금액, 결정사항)
- 사람 이름 + 관계/약속 질문

---

## 실행 예산
- 라운드: 최대 2회
- 스니펫: 최대 4~6개
- 총 pull 라인: 200줄 이하

---

## 실행 방법
1. 트리거 판단 (fast or slow?)
2. Slow면 → memory_search로 쿼리
3. 결과 스니펫만 참조 (전체 파일 read 금지)
4. 불확실하면 → 형에게 확인 질문 후 정지
