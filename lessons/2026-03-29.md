## skills/ 디렉토리 git 추적 금지

**문제**: cron 잡(이메일 요약 등)이 `git add -A` 실행 시 `skills/` 심링크를 그대로 커밋해버려서 `skills/brain-memory/` 등 실제 디렉토리가 심링크로 교체됨. 이후 `git pull`/`reset --hard` 시 복구한 스킬 파일이 날아가는 현상 반복.

**해결**: `skills/` 디렉토리를 `.gitignore`에 추가. OpenClaw가 관리하는 디렉토리는 git으로 추적하면 안 됨.

**교훈**:
- `skills/`는 OpenClaw runtime이 관리 → git 추적 대상 아님
- cron 잡의 `git add -A`는 예상치 못한 구조 변경을 커밋할 수 있음 → 가능하면 `git add <specific-paths>`로 범위 제한할 것
- 심링크가 실제 디렉토리를 덮어쓰는 git 충돌은 `git rm -r --cached <dir>` 후 `.gitignore` 추가로 해결
