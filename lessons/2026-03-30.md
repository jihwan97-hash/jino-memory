## skills/ gitignore 제거 — 스킬 버전관리 전략 변경

**문제**: `.gitignore`에 `skills/`가 등록되어 있어서 크론이 isolated 세션에서 실행될 때 스킬 스크립트를 찾지 못함. 특히 `kb-review.js`, `digest.py` 등 커스텀 스크립트가 git에 없으니 cron이 매번 실패.

**배경**: 2026-03-29 레슨에서 `skills/`를 gitignore에 추가했는데, 당시 이유는 OpenClaw 심링크가 git add -A에 의해 덮어쓰여지는 문제였음. 하지만 그 결과로 커스텀 스크립트도 함께 추적 불가 상태가 됨.

**해결**:
- `.gitignore`에서 `skills/` 라인 제거
- `skills/` 전체 git 추적 재개
- 심링크 충돌은 `git add -f` + 주의해서 커밋하는 방식으로 대응

**교훈**:
- OpenClaw 기본 스킬(~/.nvm/... 경로)과 커스텀 스크립트(skills/ 내)를 구분해야 함
- 커스텀 스크립트는 반드시 git 추적 대상 — 없으면 cron isolated 세션에서 못 찾음
- `git add -A`가 문제라면 gitignore보다 cron 페이로드의 `git add <specific-paths>` 전환이 더 나은 해결책

## daily-digest 스킬 신규 생성

**사실**: `daily-digest` 스킬이 cron에 등록되어 있었지만 `digest.py` 파일이 존재하지 않았음. `.gitignore`로 인해 생성했어도 유실되었거나, 애초에 만들어진 적 없었을 가능성.

**해결**: `skills/daily-digest/scripts/digest.py` 직접 작성
- RSS 기반 팟캐스트 수집 (Lex Fridman, a16z, Tim Ferriss)
- Brave API 키 있으면 트위터 웹 검색 보완
- 출력: `jino-memory/research/podcasts/YYYY-MM-DD-digest.md`

**교훈**: cron에 등록된 스크립트 경로는 실제 파일 존재 여부를 주기적으로 확인할 것. 파일이 없으면 cron은 조용히 실패하거나 에이전트가 직접 웹에서 수집하는 방식으로 fallback함.

## kb-review.js 신규 생성

**사실**: `kb-review` cron이 `skills/brain-memory/scripts/kb-review.js`를 실행하도록 설정되어 있었지만 해당 파일이 없었음. GitHub openclaw 레포에도 존재하지 않음 — 완전 커스텀 스크립트.

**해결**: ClawVault daily 파일(facts/decisions/goals/inbox)을 스캔하여 신규 항목 추출 + confidence/importance 기반 티어(HOT/WARM/COOL) 분류하는 스크립트 직접 작성.

**교훈**: GitHub에 없는 스크립트를 cron에 등록할 때는 반드시 git 추적 + 스크립트 생성을 함께 진행할 것.
